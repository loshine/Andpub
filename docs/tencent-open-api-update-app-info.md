# 腾讯应用开放平台 API 更新应用信息接入文档

来源：https://wikinew.open.qq.com/index.html#/iwiki/4015262492  
整理时间：2026-05-15

## 1. 能力范围

腾讯应用开放平台提供后台 API，用于更新已上架应用的版本和基础资料。接入后可以不登录管理中心，直接由服务端完成应用信息、安装包、图标、截图、版权材料、资质材料、视频等内容的提交。

限制：

- 只支持更新已有应用，不支持新应用发布。
- 只支持开发者主账号调用，子账号不支持。
- 调用前，应用必须已在开放平台接入并成功发布上线。
- 更新会进入平台审核；审核通过后按提交时的发布策略发布。

## 2. 接入准备

1. 在腾讯应用开放平台找到目标 Android 应用，确认应用已上线。
2. 在应用管理页进入：`账户管理` -> `API发布接口`。
3. 点击申请开通，获取 `access_secret`。
4. 准备调用所需的 `user_id`、`app_id`、`pkg_name`。

## 3. 调用规则

服务地址：

| 环境 | 地址 |
| --- | --- |
| 正式环境 | `https://p.open.qq.com/open_file/developer_api` |

通用约定：

- 协议：HTTP。
- 方法：各接口均为 `POST`。
- 默认 `Content-Type`：`application/x-www-form-urlencoded`。
- 请求与响应编码：`UTF-8`。
- URL 中的参数名、参数值需要 URL 编码。
- `application/x-www-form-urlencoded` 的 body 参数值也需要 URL 编码。
- 计算签名时，参数名和参数值不做 URL 编码。

公共参数：

| 参数 | 类型 | 必传 | 说明 |
| --- | --- | --- | --- |
| `user_id` | string | 是 | 开发者账号的 UserID |
| `timestamp` | string | 是 | 秒级时间戳，例如 `1724043864`；客户端时间误差最大 30 分钟 |
| `sign` | string | 是 | HMAC-SHA256 签名结果 |

签名规则：

1. 将公共参数和业务参数合并。
2. 排除 `sign` 本身；值为 `null` 的参数不参与签名。
3. 按参数名 ASCII 升序排序。
4. 拼成 `k1=v1&k2=v2` 字符串。
5. 使用 `access_secret` 作为 key 计算 HMAC-SHA256。
6. 将 hash 结果转为小写 16 进制字符串，作为 `sign`。

文档原文里有一处写成“除 `api_sign` 外”，但公共参数实际是 `sign`。实现时按 `sign` 处理，别把签名字段自己签进去。

## 4. 推荐调用流程

有文件更新时：

1. 调用 `/query_app_detail` 查询当前应用资料。
2. 对需要上传的文件逐个调用 `/get_file_upload_info`，拿到 COS 预签名 URL 和文件流水号。
3. 使用预签名 URL 上传文件到 COS。
4. 调用 `/update_app`，提交基础信息和文件流水号。
5. 调用 `/query_app_update_status` 查询审核状态。

只更新基础信息时：

1. 调用 `/query_app_detail` 查询当前应用资料。
2. 调用 `/update_app`，只传需要变更的字段。
3. 调用 `/query_app_update_status` 查询审核状态。

## 5. 查询应用详情

接口：`POST /query_app_detail`

用途：按 `app_id` 和包名查询应用详情。调用更新接口时，可以把查询结果中的可更新字段按需原样传入或变更后传入。

请求参数：

| 参数 | 类型 | 必传 | 说明 |
| --- | --- | --- | --- |
| `pkg_name` | string | 是 | 应用包名 |
| `app_id` | string | 是 | 应用 ID |

响应参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `ret` | int | 返回码；`0` 表示成功 |
| `msg` | string | 返回描述；失败时带错误说明 |
| `pkg_name` | string | 应用包名 |
| `app_name` | string | 应用名称 |
| `app_type` | int | 应用类型：`1` 软件，`2` 不联运游戏，`3` 联运网游，`4` 联运单机，`5` 联运免费 |
| `category` | int | 应用分类 |
| `operator` | string | 应用运营方 |
| `developer` | string | 应用研发方 |
| `introduce` | string | 应用简介 |
| `one_word_summary` | string | 一句话简介 |
| `age_level` | int | 年龄分级：`3`、`8`、`12`、`16`、`18` |
| `screen_size` | int | 屏幕尺寸：`0` 全部，`1` 480x320 以上，`2` 640x480 以上，`3` 960x720 以上 |
| `language` | int | 语言：`0` 中文，`1` 英文，`2` 多语言 |
| `is_support_ipv6` | int | IPv6：`0` 支持，`1` 不支持 |
| `device_type` | int | 设备：`0` Phone，`1` Pad，`2` Phone 和 Pad |
| `feature` | string | 版本特性说明 |
| `login_flag` | int | 是否涉及登录：`1` 是，`2` 否 |
| `login_account` | string | 测试登录账号；涉及登录时必填 |
| `pay_type` | int | 支付类型：`1` 无支付无提现，`2` 无支付仅提现，`3` 含支付 |
| `demo_video_flag` | int | 是否涉及配套设备：`1` 是，`2` 否 |

## 6. 文件上传

文件上传分两步：

1. 调用 `/get_file_upload_info` 获取 COS 预签名 URL 和文件流水号。
2. 使用预签名 URL 上传文件。之后在 `/update_app` 中传对应流水号。

### 6.1 获取上传信息

接口：`POST /get_file_upload_info`

限制：每个用户每天最多调用 100 次，即最多申请 100 个文件上传。

请求参数：

| 参数 | 类型 | 必传 | 说明 |
| --- | --- | --- | --- |
| `pkg_name` | string | 是 | 应用包名 |
| `app_id` | string | 是 | 应用 ID |
| `file_type` | string | 是 | 文件类型：`img` 图片，`apk` 安装包，`pdf` PDF，`video` 视频，`txt` 文本 |
| `file_name` | string | 是 | 带后缀文件名，例如 `icon.png`、`app.apk`、`license.pdf`、`demo.mp4` |

响应参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `ret` | int | 返回码；`0` 表示成功 |
| `msg` | string | 返回描述 |
| `pre_sign_url` | string | COS 预签名 URL |
| `serial_number` | string | 文件上传流水号 |

上传注意事项：

- 用 `PUT` 请求把文件内容上传到 `pre_sign_url`。
- 建议同步读取完整文件后再上传。
- 文件上传可能较慢，客户端超时时间建议设置为 1 分钟以上，按文件大小调整。
- COS 上传失败时参考腾讯云 COS 预签名上传、异常处理、错误码文档排查。

## 7. 更新应用

接口：`POST /update_app`

用途：更新应用基础信息和文件信息。可以只更新基础信息，不传文件流水号。

限制：

- 每个用户每天最多调用 50 次。
- 字符串参数不要包含 emoji。
- 接口处理耗时可能较长；不涉及 APK 时建议超时至少 15 秒，涉及 APK 时建议 60 秒以上。

请求参数：

| 参数 | 类型 | 必传 | 说明 |
| --- | --- | --- | --- |
| `distribution_end` | string | 否 | 分发终端选项，按 `PC端/车载端/平板端/其他智能设备端` 排列的二进制字符串，例如 `1001`；不传或空字符串则沿用上次选择 |
| `pkg_name` | string | 是 | 应用包名 |
| `app_id` | string | 是 | 应用 ID |
| `app_name` | string | 否 | 应用名称；1 个自然年最多修改 2 次，修改后需要重新上传软著 |
| `modify_app_name_reason` | string | 否 | 修改应用名称的原因 |
| `category` | int | 否 | 应用分类；`31003`、`35002`、`35004` 禁止变更 |
| `modify_category_reason` | string | 否 | 修改分类的原因 |
| `operator` | string | 否 | 应用运营方 |
| `developer` | string | 否 | 应用研发方 |
| `introduce` | string | 否 | 应用简介 |
| `one_word_summary` | string | 否 | 一句话简介 |
| `age_level` | int | 否 | 年龄分级：`3`、`8`、`12`、`16`、`18` |
| `icon_file_serial_number` | string | 否 | 图标文件流水号；512x512、200KB 以内、PNG、直角图标 |
| `snapshots_file_serial_number` | string | 否 | 截图文件流水号；多张用 `\|` 分隔，建议 4 到 5 张，1080x1920，最低 320x480，JPG/PNG，单张不超过 1MB，所有图片宽高一致 |
| `screen_size` | int | 否 | 屏幕尺寸：`0` 全部，`1` 480x320 以上，`2` 640x480 以上，`3` 960x720 以上 |
| `language` | int | 否 | 语言：`0` 中文，`1` 英文，`2` 多语言 |
| `ipv6` | int | 否 | IPv6：`0` 支持，`1` 不支持 |
| `device_type` | int | 否 | 设备：`0` Phone，`1` Pad，`2` Phone 和 Pad |
| `feature` | string | 否 | 版本特性说明 |
| `deploy_type` | int | 是 | 审核通过后的发布方式：`1` 立即发布，`2` 定时发布 |
| `deploy_time` | int | 否 | 定时发布时间；`deploy_type=2` 时必填，秒级时间戳，北京时间，必须晚于当前时间 |
| `copyright_elec_cert_file_serial_number` | string | 否 | 电子版权证书流水号；1 份 5MB 以内 PDF |
| `copyright_licences_file_serial_number` | string | 否 | 软著版权证明流水号；1 到 10 张，JPG/PNG，单张 10MB 以内，多张用 `\|` 分隔 |
| `is_soft_delegation` | int | 否 | 软著资质是否包含转授权：`0` 否，`1` 是 |
| `soft_delegation_file_serial_number` | string | 否 | 转授权材料流水号；`is_soft_delegation=1` 时按平台要求上传授权书和授权方证明材料 |
| `soft_delegation_period_type` | int | 否 | 转授权有效期类型：`0` 指定有效期，`1` 永久有效；`is_soft_delegation=1` 时需要 |
| `soft_delegation_start_time` | int | 否 | 转授权开始时间；秒级时间戳，北京时间；指定有效期时需要 |
| `soft_delegation_end_time` | int | 否 | 转授权结束时间；秒级时间戳，北京时间；指定有效期时需要 |
| `special_ind_category` | string | 否 | 特殊行业分类 JSON 字符串；格式见下文 |
| `other_copyright_file_serial_number` | string | 否 | 其他版权或资质证明流水号；多张用 `\|` 分隔 |
| `security_reports_file_serial_number` | string | 否 | 安全评估报告和安全承诺书流水号；多张用 `\|` 分隔 |
| `apk32_flag` | int | 否 | 是否上传 32 位或 32/64 兼容包：`1` 是，`2` 否；为 `1` 时必须传 `apk32_file_serial_number` |
| `apk64_flag` | int | 否 | 是否上传 64 位包：`1` 是，`2` 否；为 `1` 时必须传 `apk64_file_serial_number` |
| `apk32_file_serial_number` | string | 否 | 32 位或 32/64 兼容包流水号 |
| `apk32_file_md5` | string | 否 | 32 位或 32/64 兼容包 MD5；传了 `apk32_file_serial_number` 时必填 |
| `apk64_file_serial_number` | string | 否 | 64 位包流水号 |
| `apk64_file_md5` | string | 否 | 64 位包 MD5；传了 `apk64_file_serial_number` 时必填 |
| `login_flag` | int | 否 | 是否涉及登录：`1` 是，`2` 否 |
| `login_account` | string | 否 | 测试账号；`login_flag=1` 时必填，格式建议：`账号：xxx 密码：xxx` |
| `pay_type` | int | 否 | 支付类型：`1` 无支付无提现，`2` 无支付仅提现，`3` 含支付 |
| `pay_promise_file_serial_number` | string | 否 | 无支付仅提现承诺函流水号；`pay_type=2` 时需要 |
| `demo_video_flag` | int | 否 | 是否涉及配套设备：`1` 是，`2` 否 |
| `demo_video_file_serial_number` | string | 否 | 配套设备操作视频流水号；`demo_video_flag=1` 时需要，视频 500MB 以内，MP4/M4A |

响应参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `ret` | int | 返回码；`0` 表示成功 |
| `msg` | string | 返回描述 |

```json
{
```

- `status` 固定为 `true`。
- `checkedKeys` 支持多个值。

## 8. 查询审核状态

接口：`POST /query_app_update_status`

用途：查询已提交应用更新的审核状态。

请求参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pkg_name` | string | 应用包名 |
| `app_id` | string | 应用 ID |

响应参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `ret` | int | 返回码；`0` 表示成功 |
| `msg` | string | 返回描述 |
| `audit_status` | int | 审核状态：`1` 审核中，`2` 审核驳回，`3` 审核通过，`8` 开发者主动撤销 |
| `audit_reason` | string | 审核原因说明 |

## 11. 实现检查清单

- `timestamp` 使用秒级 Unix 时间戳，并保证机器时间准确。
- 签名时排除 `sign` 字段，按 ASCII 升序拼接。
- body 使用 `application/x-www-form-urlencoded`。
- 文件先申请上传信息，再上传到 COS，最后把流水号传给 `/update_app`。
- 只传本次需要变更的字段；不变更的可不传。
- 涉及名称、分类、软著、特殊行业、支付、登录、配套设备时，提前准备审核材料。
- `/update_app` 成功只代表提交成功，不代表审核通过；必须轮询 `/query_app_update_status`。
